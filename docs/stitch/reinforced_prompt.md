Create a mobile-first Korean app concept for “AI 첫걸음”.

This must be designed as an AI Agent product, not a simple AI chatbot, not a generic assistant, not a lecture platform, and not an LMS.

Core definition:
“AI 첫걸음” is a 생활 개입형 교육 AI Agent.
It intervenes in real-life situations where users feel confused, intimidated, or blocked by digital/AI-related tasks.
It does not simply answer questions.
It must:
- understand the user’s situation
- classify the problem type
- decide the most helpful next action
- guide the user step by step
- generate practical outputs the user can immediately use
- explain why in very simple Korean
- turn the solved situation into a reusable learning moment

The service should feel like:
“문제를 대신 풀어주는 챗봇”이 아니라
“상황을 파악하고, 해결 경로를 제시하고, 실행까지 돕고, 다음에는 스스로 할 수 있게 만드는 Agent”

Primary target users:
- Korean adults age 40–70
- older adults with low AI confidence
- digital beginners
- parents who are unfamiliar with AI tools
- non-technical office workers
- family caregivers who help older family members

Main user insight:
These users do not want to open an app just to study.
They will only use the product if it solves an immediate real-life problem with low friction.
Therefore, the product must feel like an everyday life problem-solving agent first, and a learning agent second.

Main real-life use cases:
- understanding difficult public service messages
- understanding hospital or bank notifications
- interpreting screenshots or photos of notices
- checking suspicious scam-like messages
- preparing questions before phone calls
- rewriting difficult or emotional family messages into clearer communication
- writing polite replies
- simplifying official or confusing text into easy Korean
- helping users understand what to do next in practical situations

Critical product principles:
1. Agentic behavior must be visible in the UI
The app must clearly show:
- context understanding
- problem classification
- next-action recommendation
- adaptive explanation
- reusable learning feedback

2. Low-friction input
Use:
- photo upload
- screenshot upload
- voice input
- short pasted text
Avoid requiring complicated manual copy-paste flows.

3. Immediate usefulness
The app should solve the problem first.
Then it should say:
- what happened
- what this means
- what to do next
- how to ask similarly next time

4. Learning through action
The learning experience should not feel like a school lesson.
It should feel like:
“I solved my real problem, and now I understand how to do it again.”

5. Accessibility
Must support:
- large text
- high contrast
- simple buttons
- one primary action per screen
- easy Korean
- voice guidance
- minimal clutter

I want the app concept to communicate the following product differentiation:

This is not a chatbot because:
- it does not wait passively for open-ended prompts only
- it actively identifies the situation type
- it proposes next actions
- it guides the user through a structured resolution flow

This is not an LMS because:
- it does not mainly provide course content
- it teaches through real-life intervention and adaptive support

This is not a general AI assistant because:
- it is specifically designed for digital beginners and low-confidence users
- it prioritizes accessibility, reassurance, step-by-step execution, and AI literacy growth

Please generate the following app screens:

1. Splash / Intro screen
- App name: AI 첫걸음
- Tagline:
  “생활 속 어려움에 바로 개입하는 교육 AI Agent”
- Warm, trustworthy, non-technical visual tone
- CTA: “바로 시작하기”
- Secondary CTA: “어떤 Agent인가요?”

2. Home screen
This is the most important entry point.
It must feel like immediate problem-solving, not education.
Primary action cards:
- 문자/안내문 이해하기
- 사진으로 쉽게 설명받기
- 답장 문장 만들기
- 전화 전 질문 정리하기
- 사기 문자 확인하기
- 음성으로 바로 물어보기

Home should also show:
- Agent recommendation area
  Example:
  “방금 올린 내용은 병원 안내문 유형입니다.”
  “지금 필요한 건 쉬운 설명 + 질문 정리입니다.”
- simple recent activity summary
- “쉬운 설명 모드” toggle

3. Input screen
Allow the user to choose:
- 사진 올리기
- 스크린샷 올리기
- 문자 붙여넣기
- 음성으로 말하기
Very simple UI with large buttons.

4. Agent analysis screen
This must visually show agent behavior.
After input, the screen should show:
- detected situation type
  examples:
  공공기관 안내 / 병원 안내 / 금융 안내 / 가족 대화 / 의심 문자 / 일반 문의
- confidence or clarity indicator
- what the Agent is doing
  examples:
  “내용을 쉽게 풀어쓰는 중”
  “지금 해야 할 행동을 정리하는 중”
  “필요한 질문을 준비하는 중”
- recommended flow selection
  examples:
  쉬운 설명 보기 / 바로 답장 만들기 / 행동 단계 정리 / 전화 질문 준비

This screen is important because it must prove this is an agent, not a chatbot.

5. Result screen
This should include:
- 한 줄 요약
- 쉬운 설명
- 지금 해야 할 일
- 바로 쓸 수 있는 문장
- 복사 / 공유 버튼
- 위험 여부 or 중요도 if relevant
- “다음엔 이렇게 물어보세요” reusable learning tip

6. Guided action screen
A structured step-by-step execution screen.
Examples:
- 질문 리스트 만들기
- 전화 전에 읽을 문장 정리
- 답장 문장 선택하기
- 위험 문자 차단 행동 안내
- 공식 안내문 쉽게 이해하기

This screen must show:
- one step at a time
- reassurance
- simple confirmation buttons
- “더 쉽게 설명해줘”
- “예시 보여줘”
- “그냥 바로 써줄래?”

7. Learning transition screen
After solving the problem, the app should transform the solved case into a learning moment.
Show:
- 오늘 해결한 문제
- 오늘 배운 표현
- 다음에 혼자 물어볼 수 있는 방식
- 비슷한 상황 다시 해보기
This should feel light and practical, not academic.

8. Progress / My page
Keep it simple.
Show:
- 해결한 생활 문제 기록
- 자주 도움받은 유형
- 점점 익숙해지고 있는 기능
- 추천 다음 단계
Do not make it feel like a student dashboard.

9. Accessibility settings screen
- 큰 글씨
- 음성 안내
- 쉬운 설명
- 단계별 진행 강화
- 고대비 모드

10. About / Service explanation screen
Clearly explain:
- this is an AI Agent, not a chatbot
- it understands context, classifies situations, recommends actions, supports execution, and turns solved moments into learning
Use friendly visual cards.

Visual direction:
- mobile-first
- Korean app style
- accessible and calm
- large rounded cards
- warm but modern
- family-friendly and senior-friendly
- clean icons
- high readability
- emotionally safe
- avoid futuristic AI styling
- avoid cyberpunk
- avoid developer dashboard
- avoid SaaS complexity
- avoid classroom/LMS look

Color direction:
- soft blue
- warm white
- muted green
- subtle yellow highlights
- calm, safe, trustworthy

Very important:
The app must visually and structurally communicate “agentic intervention”.
I want to see decision points, classification, recommendations, next steps, and adaptive guidance.
Do not design this as a generic chat screen.
Do not over-focus on conversation bubbles.
Focus on guided action flows and agent reasoning made understandable to ordinary users.

Output:
Generate a polished, demo-day ready Korean mobile app UI concept that feels original, practical, accessible, emotionally convincing, and clearly agent-based.
