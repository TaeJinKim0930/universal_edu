Create a mobile-first web app UI/UX concept for an AI education service called "AI 첫걸음 (AI First Step)".

This is not a generic chatbot app, not a lecture platform, and not a simple LMS.
It is an AI learning agent for people who are unfamiliar with AI, especially middle-aged users, older adults, digital beginners, and non-technical adults in Korea.

Core concept:
AI 첫걸음 helps people who do not know how to use ChatGPT or AI tools actually learn by doing small real-life tasks.
The service lowers the barrier to entry for AI learning.
It should feel extremely easy, warm, safe, friendly, and guided step-by-step.
The app should look trustworthy, accessible, and emotionally supportive, not technical or intimidating.

Target users:
- Adults in their 40s to 70s
- Older adults who can use smartphones but are not comfortable with new technology
- Office workers who want to start using AI in daily tasks
- Digital beginners who feel AI is difficult

Main UX direction:
- Very simple interface
- Large buttons
- Large readable typography
- High contrast but soft, friendly color palette
- Step-by-step guided flow
- Voice input friendly UI
- Minimal clutter
- Emotional reassurance and clear next action on every screen
- Korean-first UX
- Feels like a learning coach, not a productivity dashboard

Brand tone:
- Warm
- Trustworthy
- Encouraging
- Easy
- Human-centered
- Practical
- Calm and non-judgmental

Design keywords:
accessible, simple, guided, senior-friendly, educational, conversational, assistant-like, Korean mobile app, warm onboarding, practical life-learning

Please design the following pages/screens:

1. Splash / Intro screen
- App name: AI 첫걸음
- Short tagline:
  "AI를 모르는 사람도, 오늘부터 쉽게"
- Friendly illustration of an older adult or beginner learning with AI
- Primary CTA: "시작하기"
- Secondary CTA: "어떤 서비스인가요?"

2. Onboarding flow (3-4 screens)
Purpose: show that this app is for beginners and explain how it works
Should communicate:
- You do not need to know AI already
- The app teaches through real-life tasks
- The app adapts to your level
- Voice and simple mode are available
Include progress indicator and large navigation buttons

3. Simple level diagnosis screen
Purpose: identify user skill level and preferred learning style
Questions like:
- Have you used ChatGPT before?
- What do you want help with?
  Options: 문자 답장, 정보 검색, 기사 요약, 메일 작성, 일상 질문 정리
- Which mode do you prefer?
  Options: 쉬운 설명, 음성 안내, 큰 글씨, 단계별 따라하기
This should feel easy and non-technical

4. Home dashboard
This is the main entry screen after onboarding
Show:
- Welcome message
- Today’s recommended mission
- Learning progress summary
- Quick mission cards
- Recent completed tasks
- Voice start button
- “쉬운 모드” toggle
Mission examples:
- 문자 답장 도와주기
- 기사 쉽게 요약하기
- 병원/관공서 질문 정리하기
- 정중한 메일 쓰기
- 가족에게 메시지 쓰기
Make this screen feel very simple and motivating

5. Mission detail screen
This is the core educational flow
When a user selects a mission, show:
- Mission title
- Why this mission is useful in real life
- Difficulty level
- Estimated time
- Step-by-step guided learning flow
- Large next button
- Voice assist option
- Example prompts in very simple Korean
This screen must feel like a guided lesson, not a chat conversation

6. Interactive guided learning screen
This is where the AI Agent guides the user
Include:
- Step-by-step cards
- One action per step
- User input field
- Voice input button
- AI hint/reassurance area
- “다시 쉽게 설명해줘” button
- “예시 보여줘” button
- Progress indicator
This is the most important screen
It should clearly show that the service is an agent that guides, adapts, and teaches

7. Result / completion screen
Show:
- Final output result
- Copy/share buttons
- “이렇게 배웠어요” summary
- “다음엔 이렇게 말해보세요” reusable prompt tip
- Recommended next mission
This screen should connect learning and real-life application

8. My progress screen
Show:
- Completed missions
- Skill growth
- Preferred mission types
- Recommended next level
- Encouraging messages
Keep it simple, visual, and friendly

9. Accessibility / easy mode settings screen
Show easy settings such as:
- 큰 글씨
- 음성 안내
- 쉬운 설명 모드
- 단계별 안내 강화
- 고대비 모드
This is important because accessibility is one of the core concepts

10. Service introduction / about screen
Explain:
- Why this app exists
- Who it is for
- How it is different from a normal chatbot or lecture platform
Use clear cards or sections
Mention:
“AI 첫걸음은 답만 주는 서비스가 아니라,
사용자의 수준을 파악하고, 생활 속 문제를 해결하면서 자연스럽게 AI를 익히도록 돕는 학습형 AI Agent입니다.”

Important product differentiation:
This service is different from:
- LMS: because it does not just provide content
- Chatbot: because it does not just answer
- AI tutor: because it does not only explain concepts
Instead, it diagnoses the user’s level, recommends a mission, guides step-by-step, adapts explanations, and helps the user reuse what they learned in real life

Style requirements:
- Mobile-first responsive layout
- Modern Korean app aesthetic
- Friendly rounded cards
- Soft shadows
- Clean iconography
- Minimal cognitive load
- Strong information hierarchy
- Very readable typography
- Avoid overly futuristic AI style
- Avoid dark cyberpunk visuals
- Avoid developer-tool dashboard feeling
- Avoid cluttered enterprise SaaS feeling

Color direction:
Use warm and safe tones, such as soft blue, warm white, gentle green, muted yellow accents
The experience should feel hopeful, approachable, and educational

Output request:
Generate a polished app UI concept with all major screens connected in one consistent product system.
Make it look like a real startup product ready for demo day.
Focus on emotional trust, accessibility, and guided learning.
